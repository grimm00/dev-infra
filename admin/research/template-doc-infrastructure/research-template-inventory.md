## ğŸ“‹ Requirements Discovered

### Functional Requirements

- **FR-1:** System must provide templates for all 17 identified doc types
- **FR-2:** Templates must include all required sections with placeholder markers
- **FR-3:** Templates must use consistent status header format across all types
- **FR-4:** Hub templates must include Quick Links section with standardized format
- **FR-5:** Templates must support two-mode generation (scaffolding vs full)
- **FR-6:** Templates must use consistent priority/status indicators
- **FR-7:** Each workflow must have a hub template (README.md variant)

### Non-Functional Requirements

- **NFR-1:** Template structure must be parseable for validation
- **NFR-2:** Templates should enable line count validation
- **NFR-3:** Template naming must follow kebab-case convention
- **NFR-4:** Templates must be self-documenting (comments explain usage)

### Constraints

- **C-1:** Templates must remain pure markdown (no custom syntax)
- **C-2:** Templates must work with existing command structures
- **C-3:** Line counts are guidance, not hard limits
