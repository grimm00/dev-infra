This exploration clarifies proj-cli's focused role:

**What proj-cli IS:**
- Project lifecycle management (init, plan, apply, sync)
- State tracking via `.dev-infra.yml`
- Registry management via work-prod API

**What proj-cli is NOT:**
- Developer tooling (that's dev-toolkit)
- Data storage (that's work-prod)
- Template source (that's dev-infra)
