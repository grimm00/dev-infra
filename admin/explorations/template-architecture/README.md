## ðŸŽ¯ Overview

The current `standard-project` template uses a simple `frontend/` + `backend/` directory structure. This works for basic full-stack web apps but may be too simplistic for:

- API-only services (no frontend)
- CLI tools (no traditional backend/frontend split)
- Microservices (multiple services)
- Libraries/packages (no runtime)
- Projects with infrastructure code

This exploration investigates alternative approaches to template structure that better accommodate the diversity of real-world projects.
