## ðŸŽ¯ Overview

This exploration investigates evolving dev-infra from a "template factory" to a "project infrastructure manager" - a system that maintains an ongoing relationship with the projects it creates, similar to how Terraform manages infrastructure.

**Key Concepts:**
- **State Tracking:** `.dev-infra.yml` tracks what version/config each project has
- **Plan/Apply Workflow:** Preview changes before applying them
- **CI/CD Push Updates:** Automatically create PRs when templates change
- **Drift Detection:** Know when projects have diverged from templates
